9 # C++ Backport
10 
11 **Backport** is an ongoing effort to bring modern C++ utilities to be compatible
12 with C++11.
13 
14 It is a header-only library that implements much of the functionality from
15 C++14, C++17, and C++20 to be compatible with C++11. `constexpr` support is
16 conditionally enabled based on the language setting of the compiler (e.g.
17 if using C++14, more `constexpr` is available to the user than with C++11)

36 ## Rationale
37 
38 C++11 was release _9 years ago_, and yet many organizations still have yet to
39 move onto more modern releases. In some cases this is due to security reasons,
40 in others it may be that the toolchains don't yet exist for more modern C++
41 support. In any case, much of the newer C++ library types don't actually
42 require newer C++ language features -- but provide significant value to the
43 developer by creating a new vocabulary that doesn't exist in prior versions.
44 
45 Much of the newer utilities may also have parallels in **Boost** libraries,
46 however many organizations are unable to adopt this -- either due to the domain
47 being too constrained, or due **Boost** being a large dependency to introduce.
48 
49 As a result, this library exists as a standalone, **header-only** drop-in for
50 newer standard library features from C++11, C++14, C++17, and C++20.

74 ## FAQ
75 
76 ### Where is `std::filesystem`?
77 
78 This library aims to deliver a _header-only_ solution to the many missing
79 types that have been introduced in newer standard libraries. `std::filesystem`
80 is a bit complicated to handle, since it introduces a dependency on the
81 operating system being compiled against. Implementing `std::filesystem` will
82 generally require conditional compilation, and -- for best practices --
83 separate source files in order to insulate use of system headers (lest this
84 library introduce the perils of the `<windows.h>` macros). This would change the
85 library fundamentally so that it is no longer a header-only solution -- and this
86 is not desirable.
87 
88 Third-party implementations of `std::filesystem` should exist, which can be
89 independently used.
